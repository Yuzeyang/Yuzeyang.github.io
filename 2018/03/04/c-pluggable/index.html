跨平台组件的插件化实践

2018年3月4日

背景

在去年我们按照 WebSocket 协议实现了一套跨平台组件之后，最近我开始对其进行了插件化的改造，为什么要这么做呢？首先我们的结构是 Worker (负责和业务层通信) -> WebSocket -> TLS -> uv，而在这四层结构里面存在不少耦合与调用不合理的地方，使得这四层是紧密结合在一起，比如：

如果这时候我们需要剔除 TLS 或者扩展日志功能等，都会改动相当多的代码，所以插件化改造的目的一个是为了解耦，另一个是为了实现组件的可插拔性。

核心

而我在做插件化的核心思路是采用双向链表和函数指针，如果秒懂的同学可以先在脑海里想一下具体实现哈，如果还不太明白的同学那么往下看，我会详细介绍。

接口调用

以往我们对于接口的调用都是显示的，但这样是耦合的，显示的调用是因为我们知道这些接口是可用的，而在插件化中某个插件可能没有被注册，这样接口的调用会在某一层中断而导致组件的不可用，所以我们可以使用函数指针来解决这个问题。

我们在插件的初始化中把插件中函数的首地址存储在函数指针变量中

这样上层在不需要知道下一个插件的情况下，可以通过指针地址找到对应的结构体，来调用其 conn 函数。

解耦

解耦的工作就是将各插件中不属于自己的处理整理到对应插件中，再用函数指针变量将各个环节串起来。
